🎯 BUGS FOUND BY AANYA

1. ⬅️➡️ NEXT BUTTON JUMPS 2 QUESTIONS
   └─ User clicks Next and goes from Q1 → Q3 instead of Q1 → Q2
   └─ Occurs in both Topic Practice and Mock Exam

2. 📊 SCORE EXCEEDS TOTAL QUESTIONS (34/30 = 113.3%)
   └─ Final score: 34/30 (impossible!)
   └─ Percentage: 113.3% (impossible!)
   └─ Happens when user goes back and re-answers same question

───────────────────────────────────────────────────────────────────────

✅ FIXES APPLIED

FIX #1: Next Button Now Safe ✓
───────────────────────────────
PROBLEM:
  - After submitting, system auto-advances to next question
  - If user clicked Next without submitting, it advances again
  - Result: 2 questions skipped!

SOLUTION:
  - Next button is DISABLED until current question is answered
  - Help text shows: "Answer the question first"
  - Prevents skipping any questions

WHERE:
  - show_practice_mode() - Topic Practice
  - show_mock_exam() - Mock Exam

FIX #2: Score Calculation Now Correct ✓
────────────────────────────────────────
PROBLEM:
  - Score incremented each time user re-submitted same question
  - Going back to Q1 and re-submitting → Score incremented twice!
  - Result: 34 points on 30 questions = 113.3% accuracy!

SOLUTION:
  - Check if question already answered before incrementing score
  - If already answered, show "Already submitted" message
  - Score only increments once per unique question

WHERE:
  - show_practice_mode() - Topic Practice
  - show_mock_exam() - Mock Exam

───────────────────────────────────────────────────────────────────────
